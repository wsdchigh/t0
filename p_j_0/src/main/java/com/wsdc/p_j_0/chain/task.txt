                        任务链

<li>    任务链，就是在任务之间设置关联，是的彼此之间能够先后执行
        <li>    减少在代码中的沉冗
        <li>    将逻辑主干清晰的显示出来
        <li>    不会减少代码的执行性能，反而会增添一点点

        比如,我们调用照相机功能
        <li>    检测是否获取相机的运行时权限
        <li>    打开相机
        <li>    我们需要先检测权限，在明确获取权限之后，才能去打开相机

        因此，我们将检测运行时权限设置为任务A,打开相机作为一个标记设置为B，执行最终打开相机设置为C
        <li>    C只有在A，B同时满足的情况之下才能执行
        <li>    这样子，我们将所有的代码均写在了一起，这样直观，同时减少了很多判断代码

<li>    任务链是在子线程中执行的，所以，针对延时任务，不会影响主线程的渲染


<li>    缺点
        <li>    代码是聚集在一起了，但是代码聚集在一起，并不一定好看
        <li>    尽管这是链式调用
